# 🚀 Email Helper: Quick Start Guide

Цей посібник допоможе вам швидко налаштувати проект та вирішити типові проблеми при розробці на macOS.

---

## 🛠 1. Швидкий старт (Встановлення)

Виконайте ці кроки для першого запуску проекту:

### 1.1 Клонуйте репозиторій:
```bash
git clone https://github.com/misha-vynnyk/email-helper.git
```

### 1.2 Перейдіть у папку проекту:
```bash
cd email-helper
```

### 1.3 Перейдіть на гілку з останніми змінами (TTT Profile):
```bash
git fetch origin
git checkout feature/ttt-profile-integration
```

### 1.4 Встановіть залежності:
```bash
npm install
```

---

## ⚡ 2. Запуск проекту

Після встановлення ви можете запустити проект локально.

### Стандартний запуск:
```bash
npm run dev
```

### Запуск кількох інстансів (без конфліктів):
Якщо вам потрібно запустити кілька копій проекту одночасно, використовуйте ідентифікатор інстансу. Це автоматично розділить порти та профілі браузера:

*   **Другий інстанс:** `npm run dev 1`
*   **Третій інстанс:** `npm run dev 2`

> [!NOTE]
> Кожен інстанс має власну ізольовану сесію автоматизації. При першому запуску нового інстансу (наприклад, з ID 1) вам потрібно буде знову залогінитися у сховище.

---

## 🍏 3. Вирішення проблем на macOS (AppleDouble ._*)

Якщо ви бачите помилки, пов'язані з файлами `._*` у папці `.git`, скористайтеся цими командами:

### 3.1 Очищення поточних метаданих:
```bash
COPYFILE_DISABLE=1 find .git/objects/pack -name "._*" -delete
```

### 3.2 Глобальне ігнорування (рекомендовано):
Щоб Git ігнорував системні файли macOS у всіх проектах:
```bash
echo "._*" >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global
```

### 3.3 Примусове очищення теки:
```bash
dot_clean .git/objects/pack
```
