# Hidden Sections Feature

## Опис

Функція дозволяє приховувати певні секції HTML темплейтів **тільки для відображення** (preview). Код темплейту залишається незмінним при копіюванні, відправці email та завантаженні в редактор.

## Використання

### 1. Маркування секцій в HTML

Використовуйте коментарі для позначення секцій, які можна приховувати:

#### Формат 1 (рекомендований):
```html
<!--=== SectionName ===-->
<div>Контент, який можна приховати</div>
<!--=== SectionName-end ===-->
```

#### Формат 2 (альтернативний):
```html
<!--=== SectionName ===-->
<div>Контент, який можна приховати</div>
<!--=== /SectionName ===-->
```

### 2. Приклад використання

```html
<!DOCTYPE html>
<html>
<head>
  <title>Email Template</title>
</head>
<body>
  <div class="header">
    <h1>Заголовок завжди видимий</h1>
  </div>

  <!--=== Promo-content ===-->
  <div class="promo-banner">
    <h2>Спеціальна пропозиція!</h2>
    <p>Знижка 50% на всі товари</p>
    <a href="#">Детальніше</a>
  </div>
  <!--=== Promo-content-end ===-->

  <div class="main-content">
    <p>Основний контент завжди видимий</p>
  </div>

  <!--=== Footer-social ===-->
  <div class="social-links">
    <a href="#">Facebook</a>
    <a href="#">Twitter</a>
    <a href="#">Instagram</a>
  </div>
  <!--=== Footer-social-end ===-->
</body>
</html>
```

### 3. Налаштування в UI

1. Відкрийте **Preview Settings** (кнопка налаштувань ⚙️ у Template Library)
2. Прокрутіть до секції **"Hidden Sections (Preview Only)"**
3. Введіть назву секції (наприклад, `Promo-content`)
4. Натисніть **"Add"** або Enter
5. Збережіть налаштування

### 4. Управління секціями

- **Додати секцію**: Введіть назву та натисніть "Add"
- **Видалити секцію**: Натисніть ❌ на чіпі з назвою секції
- Налаштування зберігаються автоматично в localStorage

## Важливо

✅ **Приховується тільки в preview:**
- В картці темплейту
- В повному preview діалозі

❌ **НЕ приховується:**
- При копіюванні коду (Copy button)
- В CodeMirror редакторі (View Code)
- При відправці email
- При завантаженні в редактор (Load)

## Переваги

1. **Гнучкість**: Швидко тестуйте темплейти з/без промо-контенту
2. **Безпека**: Оригінальний код залишається незмінним
3. **Зручність**: Перемикайте видимість секцій без редагування файлів
4. **Підтримка вкладеності**: Можна приховувати декілька секцій одночасно

## Приклади назв секцій

- `Promo-content` - промо-банери
- `Header-announcement` - оголошення в хедері
- `Footer-social` - соціальні мережі у футері
- `Sidebar-ads` - реклама в сайдбарі
- `Testing-section` - тестові блоки

## Рекомендації

1. Використовуйте зрозумілі назви секцій (без пробілів)
2. Називайте секції консистентно в усіх темплейтах
3. Додавайте коментарі для важливих секцій:
   ```html
   <!-- Промо-банер для акційних пропозицій -->
   <!--=== Promo-content ===-->
   ...
   <!--=== Promo-content-end ===-->
   ```

## Технічні деталі

- **Регулярні вирази**: Підтримка спецсимволів у назвах (автоматичне екранування)
- **Case-insensitive**: Регістр НЕ враховується (`Promo-content` = `promo-content`)
- **Очищення**: Автоматичне видалення зайвих порожніх рядків після видалення секцій
- **Кешування**: Налаштування зберігаються в localStorage
