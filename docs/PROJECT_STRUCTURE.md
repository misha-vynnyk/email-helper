# Project Structure

This document provides a detailed breakdown of the folders and files in the FlexiBuilder Pro Editor.

## ðŸ“‚ Root Directory

- **`automation/`**: Contains scripts and utilities for the Playwright-based storage upload automation.
- **`dist/`**: The production build output directory (generated by `npm run build`).
- **`docs/`**: This documentation folder.
- **`public/`**: Static assets served directly by Vite (e.g., `vite.svg`).
- **`server/`**: The Express.js backend for proxying requests, handling file uploads, and AI integration.
- **`src/`**: The main source code for the React application.
- **`html-converter-memory/`**: (If present) Local storage for recent conversion history or state.
- **`index.html`**: The entry point HTML file for Vite.
- **`vite.config.ts`**: Configuration for the Vite build tool.

## ðŸ“‚ `src/` Directory

The source code is organized by feature and type:

### Key Feature Modules

- **`htmlConverter/`**: **Core Feature**. Contains all logic and UI for the HTML-to-Table converter.
  - **`components/`**: UI sub-components (panels, dialogs).
  - **`hooks/`**: Custom hooks managing complex logic (e.g., `useHtmlConverterLogic`).
  - **`utils/`**: Helper functions specific to HTML conversion.
  - **`formatter.ts`**: The main transformation engine.
  - **`templates.ts`**: HTML snippets used during conversion.
  - **`StorageUploadDialog.tsx`**: Component for the "Upload to Storage" feature.

- **`imageConverter/`**: Logic for the Image Optimizer feature.
  - **`components/`**: UI for image settings and comparison.
  - **`workers/`**: Web Workers for heavy image processing (if applicable) to avoid freezing UI.

- **`emailSender/`**: UI and logic for sending test emails.
- **`emailValidator/`**: Tools for validating HTML against email clients.

- **`blockLibrary/`** & **`templateLibrary/`**: Features related to the Drag-and-Drop builder (Block Manager).

### Shared Directories

- **`components/`**: Reusable low-level UI components (e.g., `AlertDialog`, `ColorPicker`).
- **`contexts/`**: Global React Contexts (e.g., `ThemeContext`, `AuthContext`).
- **`config/`**: App-wide configuration files.
- **`hooks/`**: General-purpose hooks (e.g., `useLocalStorage`).
- **`theme/`**: MUI theme definitions and style overrides.
- **`types/`**: Shared TypeScript interfaces and type definitions.
- **`utils/`**: General utility functions (date formatting, string manipulation).

## ðŸ“‚ `automation/` Directory

Powered by Playwright + Brave.

- **`run-upload.js`**: The entry point script interface.
- **`scripts/`**: Actual Playwright scripts that drive the browser.
- **`config.json`**: Local configuration for browser paths and timeouts.

## ðŸ“‚ `server/` Directory

Powered by Express.js.

- **`server.js`** (or `index.js`): Main server entry point.
- **`ai/`**: Python scripts and environments for AI-powered features.
- **`routes/`**: API endpoint definitions.
